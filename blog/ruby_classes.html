Creating Classes In Ruby

February 6, 2016

Why Create Classes

Ruby has a ton of built in classes that represent real world objects. Strings represent lines of text, integers represent numbers, arrays and hashes represent containers for just about anything. Did you know that you can actually create your own classes in Ruby? Yup. Why would you want to do that? Well, what if you want to represent and behave like other real world objects in your programs? Creating custom classes is a great way to do this.

How Do I Create a New Class?

Let’s say you want to create a class that behaves like an ornery old man. First thing’s first, we have to name the class. Class names always begin with a capital letter, so we could name our class something like OrneryOldMan. Great! Name class: accomplished. Next, we have to decide how this class will initialize. Every class has a set initialize method that happens upon initialization of a new class instance. For example, when you call Hash.new to create a new hash, that hash is initialized with certain inherent characteristics. maybe we want to include some instance variables that set some parameters for our ornery guy. Instance variables are extremely useful because, unlike local variables, they can be used and called across different class methods. How about we give this guy an age and a default catch phrase. so far our code would look something like this: 

class OrneryOldMan
   				def initialize(age, catch_phrase)
      			@age = age
      			@phrase = catch_phrase
   				end

Instance variables are identifiable by their handle @. We now have these variables set upon initialization of the class so we can utilize them throughout our various class methods. Oh yeah……Methods! Now we get to create some methods within our class that allow us to make our old dude do stuff. Let’s see, what could we make him do? Well, maybe we have a method that makes him get even older. Let’s call that method ‘birthday’. Method creation inside a class works much the same as method creation outside of a class. Check it out:

def birthday
   				@age +=1
				end

Now, whenever we call .birthday on our new old guy, he ages by one year! Notice we’ve called our @age instance variable in this method. Ok, let’s make him be able to do something else. Let’s say that whenever he is engaged in conversation, he spits out his catch phrase. We’ll call this method ‘engage’: 

def engage
   				puts @phrase
				end

Great! we’ve got a whole new class that, upon initialization, has some cool ornery-old-guyish functionality. We can now call a new instance of this class and immediately set an age and catch phrase like this: 

burt = OrneryOldMan.new(77, "A nickel ain't worth a dime no more!")

When we call burt.birthday, burt ages by one year. When we call burt.engage he replies with “A nickel ain’t worth a dime no more!”. Fun stuff!!! Classes are fun!!! Have fun with Ruby classes!!!
