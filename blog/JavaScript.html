My First Week of JavaScript

So, I’ve spent the past week tinkering with the JavaScript language. I got to define some variables, create some functions, loop some loops, fumble through a few objects, and even create a simple game (very….very simple). I would say I’m pretty much a pro. Well, maybe not. But I did get to compare and contrast what I’ve learned with my working knowledge of Ruby. Most of my time with JavaScript, thus far, has been spent messing with ‘objects.’ An ‘object’ in JavaScript are very similar to a ‘hash’ in Ruby. Objects are essentially an unordered list of name-value pairs, called properties, which are stored inside a set of curly braces. The basic construct of a ruby hash is very similar. Hashes are an unordered set of key-value pairs that are also stored inside a set of curly braces. In both languages, we can use these containers to store all kinds of data types including strings, numbers, booleans, and even other nested containers of data. Many times, in the JavaScript language, objects will also contain functions, or custom blocks of code that can be invoked to perform tasks on other data types. Generally, in the Ruby language this same kind of actionable code would be stored in classes.

Let's take a look at the differences in syntax and accessibility for these container types. In Ruby, we have a few different ways to create hashes. One type is called 'hash literal' creation. That method looks something like this: 

my_hash = {"food" => "brownies", "number" => 6}

Hashes can also be created using the built in Hash.new method:

my_hash = Hash.new

This method will create a new empty hash that you can add key-value pairs to at your discretion.

Objects in the JavaScript language have two similar creation methods. The name-value syntax is slightly different, however. Here is an example of ‘object literal’ creation: 

my_hash = {
   dranks: "martinis",
   drunk: true
   }

Notice that Ruby hashes use the => operator to assign keys to their value where as JavaScript objects use the : operator to assign property names to their values. Also, take note of the fact that hash keys are surrounded in quotes but object property names generally are not.

Now, let's check out a few ways to access these data types in their respective containers. For now, we will just stick to accessing values with a given key or property name. In Ruby:

my_hash = {"groups" => 3, "teams" => 1}
my_hash["groups"]
=> 3

We can simply reference the hash name followed by the key name in square brackets and the value will be returned. In JavaScript, the same query might look like this:

my_hash = {
   groups: 3,
   teams:1
   };
my_hash.groups
=> 3

Ok! Much JavaScript discovery ahead for this noob. More exciting and informative blog posts soon….Hopefully.
